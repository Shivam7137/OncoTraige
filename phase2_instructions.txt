Phase Context & Objectives
Phase 1 (EDA) established three critical findings that directly drive every preprocessing decision in Phase 2:

Severe multicollinearity — radius_mean and perimeter_mean had VIF scores above 3,500, rendering them mathematically redundant
Class imbalance — 357 benign vs 212 malignant samples creates a bias risk toward the majority class, which is clinically dangerous in cancer triage
Scale disparity — features range from 0.05 (fractal_dimension) to 4,000+ (area_worst), which distorts distance-based models and corrupts SHAP values

Phase 2 preprocessing is not a generic cleaning routine. Every step below is directly justified by a specific EDA finding and has a direct clinical consequence for model trustworthiness.

Pipeline Overview
The full preprocessing pipeline executes in strict order to prevent data leakage and ensure methodological integrity:




Step 1 — Drop ID Column
What & Why
The dataset contains a patient ID column that is purely administrative — it carries no biological information about tumor morphology. Including it creates a risk of data leakage if the model accidentally learns to associate ID ranges with diagnosis labels.

Implementation
df = df.drop(columns=['id'])

Validation Check
Confirm shape goes from (569, 32) to (569, 31) — one column dropped (ID + diagnosis + 30 features → diagnosis + 30 features)
Confirm no null values introduced

Never drop the diagnosis column at this stage. It is the target variable, not a feature — it gets separated at the split step, not here.


Step 2 — Stratified Train/Test Split
What & Why
A random split on an imbalanced dataset risks placing too few malignant cases in the test set by chance. If the test set ends up with only 30 malignant samples instead of the expected 42, every performance metric — sensitivity, AUC, F1 — becomes unreliable.
Stratified splitting guarantees the 63/37 benign-malignant ratio is preserved identically in both train and test sets.

Implementation
from sklearn.model_selection import train_test_split

X = df.drop(columns=['diagnosis'])
y = df['diagnosis'].map({'M': 1, 'B': 0})

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.20, random_state=42, stratify=y
)

Expected Output

random_state=42 ensures reproducibility — every run of the notebook produces identical splits, which is essential for a datathon submission.


Step 3 — VIF Reduction (Completed in Phase 1)
Status: Complete
VIF analysis was conducted during EDA Phase 1 as a diagnostic tool. The iterative removal process reduced the feature space from 30 to 17 features by eliminating all features with VIF > 10.

Features Removed

17 Surviving Features
texture_mean, smoothness_mean, concave_points_mean, symmetry_mean, fractal_dimension_mean, texture_se, perimeter_se, smoothness_se, compactness_se, concavity_se, concave_points_se, symmetry_se, fractal_dimension_se, area_worst, smoothness_worst, symmetry_worst, fractal_dimension_worst

These 17 features all carry genuinely independent biological information. This is what makes SHAP explanations clinically trustworthy — each feature in the explanation represents a distinct diagnostic signal.


Step 4 — Feature Scaling
What & Why
The 17 surviving features currently operate on incompatible scales. Without scaling, models interpret larger numbers as more important — a mathematical artifact with no clinical basis.


Method: StandardScaler
StandardScaler transforms each feature to zero mean and unit variance:

z = (x − μ) / σ

Where μ is the training set mean and σ is the training set standard deviation. After scaling, every feature has mean = 0 and standard deviation = 1.

Implementation
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)   # Fit ONLY on training data
X_test_scaled  = scaler.transform(X_test)        # Transform only — never fit on test

Critical leakage rule: StandardScaler must be fit exclusively on X_train. Fitting on the full dataset or on X_test leaks test set statistics into training, invalidating all performance metrics.

Impact on SHAP
Scaling is especially critical for Logistic Regression SHAP values. Unscaled, a coefficient of 0.001 on area_worst and 2.3 on fractal_dimension_worst is uninterpretable — the magnitudes reflect units, not importance. After scaling, coefficient magnitudes directly reflect feature influence and SHAP values become clinically meaningful.


Step 5 — Class Imbalance Handling
The Clinical Stakes
A model trained on an imbalanced dataset without correction will learn a systematic bias: when uncertain, predict benign. This is the most dangerous possible failure mode for a cancer triage system — systematic under-prediction of malignancy.

A model that achieves 90% accuracy by predicting benign for every patient is clinically useless. We optimize for sensitivity (recall for malignant tumors), not raw accuracy.

Chosen Method: Class Weights
Rather than modifying the data, class weights instruct the model to penalize malignant misclassifications more heavily during training. Sklearn's balanced mode automatically calculates optimal weights:

weight(class) = n_samples / (n_classes × n_samples_in_class)


Implementation
# Applied identically to all three models
LogisticRegression(class_weight='balanced')
RandomForestClassifier(class_weight='balanced')
XGBClassifier(scale_pos_weight = 285/170)   # XGBoost uses ratio format

Why Not SMOTE?
SMOTE (Synthetic Minority Oversampling) generates synthetic malignant samples by interpolating between real ones. It's a valid technique, but on a 569-sample dataset it risks creating synthetic cases that don't reflect real biological variation. Class weights achieve the same bias correction without modifying the training data, making the approach more defensible clinically and more transparent methodologically.


Step 6 — Outlier Treatment
The Design Decision
The PCA plots identified clear extreme outliers — one benign case at PC2 = 12.5 and one malignant case at PC1 = 15. Standard preprocessing practice would be to remove them. OncoTriage AI takes a deliberately different approach.

These outliers are the real-world equivalents of the Edge Case Simulator's Frankenstein tumor profiles. Removing them would make the model less robust to exactly the unusual presentations the triage system is designed to catch.

Method: Winsorization at 1st / 99th Percentile
Instead of removing outliers, Winsorization caps extreme values at the 1st and 99th percentile of the training distribution. This reduces their distorting mathematical influence while preserving the clinical reality that unusual tumor morphologies exist and must be handled.

from scipy.stats.mstats import winsorize
import numpy as np

# Apply per feature on training data, then clip test data to same bounds
lower_bounds = np.percentile(X_train_scaled, 1, axis=0)
upper_bounds = np.percentile(X_train_scaled, 99, axis=0)

X_train_winsorized = np.clip(X_train_scaled, lower_bounds, upper_bounds)
X_test_winsorized  = np.clip(X_test_scaled,  lower_bounds, upper_bounds)

Clinical Justification
A patient with an extremely unusual tumor profile presenting at a clinic should trigger a Low Reliability score and CNB escalation — not a confident malignant or benign call. Winsorization ensures these edge cases don't distort the model's decision boundary while the reliability scoring system handles them appropriately at inference time.


Step 7 — Cross-Validation Setup
Why CV Is Non-Negotiable on This Dataset
With only 455 training samples, a single train/test evaluation is statistically fragile. Performance metrics can vary by 3–5% depending purely on which 114 samples landed in the test set. For a medical decision-support system, this variability is unacceptable.

Stratified 5-Fold Cross-Validation splits the training data into 5 equal folds, trains on 4 and validates on 1, rotates 5 times, and averages results. This gives stable, trustworthy metrics with confidence intervals.

Implementation
from sklearn.model_selection import StratifiedKFold, cross_validate

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

scoring = ['accuracy', 'roc_auc', 'recall', 'precision', 'f1']

cv_results = cross_validate(
    model, X_train_winsorized, y_train,
    cv=cv, scoring=scoring, return_train_score=True
)

What to Report

Report metrics as mean ± standard deviation across folds (e.g., Sensitivity: 0.94 ± 0.02). This communicates both performance and stability — a judge who sees tight standard deviations will trust your results more than a single number.


Step 8 — Leakage Audit
Why Leakage Is the Biggest Risk
Data leakage produces models that appear to perform exceptionally well in development but fail completely in deployment. In a datathon context, a judge who spots leakage will discount all results regardless of how impressive they look. A clinical judge will correctly identify it as a patient safety risk.

Leakage Checklist

The most common leakage mistake in datathons: fitting StandardScaler on the full dataset before splitting. Always split first, scale second.


Complete Preprocessing Pipeline
The following represents the complete, ordered pipeline ready for implementation:

X_train, X_test, y_train, y_test = train_test_split(

Phase 2 Deliverables


Every object fitted during preprocessing (scaler, bounds) must be saved and reused in the Streamlit demo. When a clinician enters new patient measurements, they must be transformed using the exact same scaler fitted on training data — not refitted on new data.


Connection to the Diagnostic Reliability Score
The preprocessing pipeline doesn't just prepare data for the model — it directly enables the Diagnostic Reliability Score:

The 17 independent VIF-surviving features ensure the reliability penalty for feature dominance is measuring genuine over-reliance, not just detecting the radius/perimeter/area triplet voting together: Feature Balance component
The scaled, winsorized training data forms the reference distribution against which new patient measurements are compared — the scaler object must be preserved for this calculation: Case Similarity component
Calibrated probabilities are only meaningful after proper scaling — unscaled Logistic Regression probabilities can be systematically miscalibrated: Prediction Certainty component
All three models trained on identical preprocessed data ensures agreement differences reflect genuine disagreement, not different data transformations: Model Agreement component
